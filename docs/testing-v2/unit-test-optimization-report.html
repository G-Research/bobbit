What changed

Bundle boundary

header-toast was split away from the app render graph so session/runtime callers no longer pull render.ts into app-session-runtime.

Git and worktree fixtures

Hotspot integration suites now use cheaper fixture shapes while keeping representative route-level coverage for behavior that depends on real repository state.

Cleanup correctness

The in-process harness tracks uncertain cleanup and keeps regression coverage for sweeps that must still run when state is uncertain.

Import deferral

Core/dom tests that only need narrow assertions defer server-heavy imports until the assertion path needs them.

Repeatable profiling workflow

Run from a checkout with dependencies installed. Keep raw artifacts under .profiles/; commit only distilled docs or reports.

Goal Command Use
Full suite timing npm run test:v2:profile-hooks -- --top 30 Primary before/after comparison for wall time, summed file runtime, residuals, and cleanup counters.
Single integration hotspot npm run test:v2:profile-hooks -- --project v2-integration tests2/integration/base-ref-api.test.ts --top 20 Focused validation while preserving the profiler's report shape.
Parse an existing JSON artifact node scripts/testing-v2/profile-hooks.mjs --from-json .profiles/testing-v2/hook-profile/<run>/vitest.json --top 30 Regenerate report.md / report.json without rerunning tests.

Vitest JSON does not provide stable per-hook timings. The profiler's file residual is directional and may include hooks, module evaluation, fixtures, and reporter overhead.

Remaining measurement caveats

  • The targeted 42.12 s run is not a substitute for a full-suite after profile.
  • Machine load affects wall-clock comparisons; use summed file runtime and residuals alongside wall time.
  • Cleanup counters are available only for integration harness paths that emit stats into BOBBIT_V2_HOOK_PROFILE_DIR.
  • Do not skip uncertain cleanup to improve timings; leak detection and conservative sweeps are part of the invariant.

Conclusion

The branch restores the bundle guard and addresses the measured git/repo, cleanup, and import hotspots without weakening coverage or leak safety. The next durable deliverable is an apples-to-apples full-suite after profile from a dependency-ready checkout.
