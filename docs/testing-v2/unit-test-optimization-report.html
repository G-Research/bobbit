What changed

Git and worktree fixtures

Repeated real-git setup was replaced or narrowed where tests were validating API/status behavior rather than git mechanics. Real worktree coverage remains for the cases that require it.

Integration harness cleanup

Dirty/versioned cleanup avoids expensive no-op sweeps and default-project resets, while final leak assertions and conservative fallback cleanup remain in place.

Validation extraction

Pure grammar and error-shape checks moved out of expensive route-level fixtures, leaving representative integration coverage for real repository behavior.

Failed-suite fixes

Branch validation includes fixes needed to keep the optimized suites green under focused and broader gate runs.

Validation evidence

Passed: implementation-gate npm run test:unit at ~533 s.
Passed: local capped unit run at ~537 s.
Passed: latest gate build/check/browser/e2e validation.
Passed: focused combined validation covering 10 files and 89 tests.

The branch did not land a Vitest server/verification graph prebundle. The safer outcome was to keep source execution and bank the fixture, cleanup, and process-spawn reductions.

Remaining opportunities

  • Continue reducing proposal-edit-api beforeAll cost, which remains a visible server-heavy setup hotspot.
  • Audit remaining real git/worktree files for cases that can reuse fixture repos or move pure semantics to cheaper unit coverage.
  • Add optional CI budget thresholds once a stable post-optimization baseline is available on the target runner class.
  • Trend integration harness cleanup stats in future hook-profile reports: snapshots, sweeps, skipped sweeps, default resets/restores, and entity deletes.

Conclusion

The branch addresses the measured hotspots without removing leak safety or real process fidelity. The strongest wins are in git/worktree-heavy files and no-op cleanup avoidance; remaining work is concentrated in server-heavy setup paths and broader trend/budget automation.
